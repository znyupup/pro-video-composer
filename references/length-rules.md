# 长度规则 · TTS / 数字人 / Hook 单段硬约束

## 核心原则

**模型对长输入的"幻觉累加"是物理现象,不是参数能调的**。
单次推理越长,后段越偏 — TTS、视频生成、数字人都一样。
通过**分段合成 + 拼接** 是绕过的唯一办法。

## T2A v2 单段长度

| 上限 | 含义 |
|---|---|
| **≤ 10-15s 音频** | 模型质量稳定区 |
| **≤ 80 字中文文本** | 对应上面的音频(speed=1.2 估算)|
| **目标 60-70 字** | 留余量,推荐值 |

超过会怎样:
- 16-20s:开始能听出后半段语气松散
- 20s+:可能复读 / 情绪漂移 / 个别字发音不准
- 30s+:大概率"AI 味"明显

## 数字人 mp4 单段长度

| 上限 | 含义 |
|---|---|
| **≤ 6s mp4** | 即梦 / Vidu / Hailuo 等数字人模型质量稳定区 |

超过会怎样:
- 7-10s:口型同步开始漂
- 10s+:面部细节扭曲、眨眼频率异常

## Hook 总长度

| 段位 | 推荐长度 | 备注 |
|---|---|---|
| Hook | 20-30s | 短平快,信息轰炸 |
| 价值阐述 | 25-35s | 抛话题 + 系列定位 |
| 教学段 | 60-90s | 主体内容 |
| 演示段 | 30-60s | 屏幕录制 |
| 对比互动 | 20-30s | 留扣 + CTA |
| 结尾 | 25-30s | 系列收官 + 留言号召 |
| **整片** | **3-5 分钟** | EP 系列上限 |

## 拆分策略

### 文本拆分(给 T2A)
1. 优先按 "。" 分
2. 长句按 "," 或 ";" 拆
3. 单段超 80 字必须拆
4. 段间无空隙(`-c copy`)
5. 衔接生硬时段间加 100ms 静音

### 数字人拆分(给即梦)
1. 跟 T2A 段对齐 — 1 个 T2A 段 = 1 个数字人 mp4
2. 上传 mp3 → 即梦 web 生成 mp4(每段 1 次)
3. ffmpeg concat 6 段数字人 → hook_dh_raw.mp4

### Hook 视觉拆分(给 Remotion + b-roll)
- 节奏:2.5-3s/cut(EP3/EP4 验证)
- 太快:1.5s/cut 信息密度过高,观众累
- 太慢:5s+/cut 失去 Hook 节奏感

## 校验脚本

```bash
# 检查 emotion_plan.json 每段时长
jq '.segments[] | select(.est_duration > 15)' out/emotion_plan.json
# 输出非空 = 有超长段,让 agent 重拆

# 检查实际生成的 mp3 段时长
for f in out/narration_segments/seg*.mp3; do
  d=$(ffprobe -v error -show_entries format=duration -of csv=p=0 "$f")
  awk -v d=$d -v f=$f 'BEGIN { if (d > 15) printf "⚠️  %s %.2fs\n", f, d }'
done
```
